项目背景

现状分析：
江西省高校在校生超过120万人，毕业生离校产生大量闲置物品
新生入学季消费集中，二手商品需求旺盛
现有主流平台缺乏校园场景化支持

痛点调研（问卷数据）：
78% 有二手交易需求但缺乏合适渠道
65% 担心校外平台交易安全问题
52% 认为运费成本高于商品本身价值
43% 希望能在同校内进行交易

需求总结：
搭建面向江西省38所高校、支持同校认证、校内流转、面对面交易的校园专属二手交易平台。

技术方案

系统架构：
用户端 — React 18 + Tailwind CSS + React Router
反向代理 — Nginx（端口5000 → /api/ → 后端）
后端服务 — Node.js + Express 4（JWT认证 · bcryptjs加密 · 购物车 · 推荐系统 · AI集成）
数据库 — MongoDB 7.0（Docker容器化 · Decimal128精确价格存储）
AI引擎 — 通义千问 Qwen-plus 大模型（商品描述生成 · 智能推荐）

部署架构：
前端 — Nginx静态文件 + 反向代理，端口5000
后端 — Node.js + Express 4，端口4000，RESTful API
数据库 — MongoDB 7.0，Docker容器化运行
部署 — Docker Compose，三容器一键启动
服务器 — 2核2G云服务器，外网可访问 freevian.top

AI集成方案：
商品描述智能生成 — Qwen-plus API，输入名称自动生成专业描述
智能推荐系统 — 规则引擎 + AI包装，个性化匹配带推理标签
商品分类推荐 — Qwen-plus API，根据名称自动匹配最合适分类

---

功能实现

核心功能模块：
用户系统 — 注册/登录/JWT认证/资料编辑，bcryptjs + JWT双保险
商品管理 — 发布/编辑/删除/图片上传，图片自动压缩 + AI辅助描述
商品列表 — 分页/搜索/筛选/排序，38校$regex模糊搜索
商品详情 — 多图展示/规格参数/库存/购买，原子操作防超卖
购物车 — 添加/修改/删除/批量结算，批量原子购买
智能推荐 — 详情页底部"猜你喜欢"，AI推理标签 + 五级漏斗
个人中心 — 发布记录/购买记录/购物车管理，身份校验 + 数据聚合

安全机制：
密码安全 — bcryptjs加密，8轮盐值
身份认证 — JWT令牌，localStorage持久化
前端校验 — 手机号正则 / 地址长度 / 价格上限
后端校验 — Mongoose验证 + 业务逻辑双重拦截
防超卖 — 原子操作 $inc + $gt 条件查询
注入防护 — MongoDB参数化查询

项目创新点

创新点一：基于大模型的智能推荐系统
接入通义千问 Qwen-plus 大模型
五级漏斗规则引擎：同类目→同校→同卖家→同校用户→最新
每个推荐结果附带AI推理标签
前端"AI智能推荐"模块打造个性化体验

创新点二：38校全覆盖的校园专属生态
江西省38所高校数据库全覆盖
学院$regex模糊搜索，输入关键词即可精准匹配
省市区三级联动地址选择器
学号+手机双验证的实名认证机制

创新点三：轻量级全栈架构+容器化部署
2核2G低配服务器稳定运行
Docker Compose三容器一键部署
Nginx反向代理，前后端分离架构
Git版本控制+持续集成

创新点四：完整的电商闭环体验
商品发布→搜索筛选→AI推荐→购物车→购买→交易记录
购物车系统支持多商品批量操作和原子结算
前后端双重数据校验保障数据一致性
Decimal128精确价格存储，杜绝浮点精度问题

现场演示
