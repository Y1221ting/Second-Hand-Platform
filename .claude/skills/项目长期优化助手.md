## 已完成优化清单 (务必记住，不要重复建议)

### 安全优化 ✅
1. **JWT 认证**: tokenVersion 机制，修改密码/封禁时自增，旧 token 自动失效
2. **密码策略**: bcrypt 12 轮，8 位以上+字母数字混合，登录统一模糊错误提示
3. **账户锁定**: 5 次失败 → 锁定 15 分钟，IP 限流 20次/15分钟(登录) 5次/小时(注册)
4. **防越权**: updateUser/deleteUser 校验 `req.user._id === userId`，updateProductById 字段白名单
5. **防用户枚举**: 注册/登录统一返回模糊错误，不透露"邮箱已注册"
6. **CORS**: 函数白名单校验 origin，credentials 支持
7. **密码不回传**: 所有 API 返回前 `delete userData.password`
8. **JWT_SECRET**: 64 字节随机 hex，启动时检测默认值则拒绝启动
9. **MongoDB 端口**: `127.0.0.1:27017` 不暴露公网
10. **违禁词过滤**: 50+ 关键词，createProduct/updateProductById 拦截
11. **body-parser 限流**: 10MB 上限

### 性能优化 ✅
1. **Docker 资源硬限**: MongoDB 384MB / Backend 256MB / 已合并 nginx
2. **MongoDB 连接池**: maxPoolSize=10, minPoolSize=2
3. **MongoDB 缓存**: wiredTigerCacheSizeGB 0.15 (约 150MB)
4. **响应压缩**: compression() 中间件
5. **MongoDB 索引**: name+description 文本索引, status+category, status+department, uploadedBy.id 等复合索引
6. **前端代码分割**: React.lazy + Suspense，低频页面按需加载
7. **日志轮转**: json-file driver, 10MB max, 3 文件保留
8. **推荐引擎**: 简化为两层（同学院 + 最新）

### 架构优化 ✅
1. **容器合并**: 3→2，nginx 并入 Express 直接 serve React build
2. **健康检查**: 3 层 (mongodb mongosh ping, backend HTTP /api/health, frontend wget)
3. **start_period**: mongodb 15s, backend 10s, frontend 5s
4. **restart: always**: 所有容器崩溃自动重启
5. **dotenv 加载顺序**: server.js 第 1 行 require("dotenv")，确保 auth.js 中 JWT_SECRET 就绪
6. **备份脚本**: backup.sh (docker exec mongodump, 7 天保留)

### 功能精简 ✅
1. **砍掉 AI**: aiService/aiController/aiRoutes 已删除，前端 AI 按钮替换为快捷模板
2. **砍掉 IM**: Conversation/ChatMessage 模型 + conversationRoutes/chatRoutes 已删除
3. **砍掉评价**: Review 模型 + reviewRoutes 已删除
4. **砍掉申诉**: Appeal 模型 + appealRoutes 已删除
5. **Decimal128→Number**: price/budget 改为 Number 类型
6. **去 session**: JWT 直接签发，去掉 activeSessions/sessionGuard

## 核心约束

### 服务器硬限制 (不可突破)
```
CPU:     2 核 (最多分配给容器 1.5 核)
内存:    2GB (系统预留 300MB，容器可用 ~1.7GB)
磁盘:    假定 40GB (图片 + 数据库 + 日志)
带宽:    假定 1-5 Mbps (校园网)
```

### 设计铁律
1. **任何方案必须适配 2核2G**，如果方案需要加服务器，必须标注"需要扩容"
2. **不做大厂功能拷贝**，校园场景优先微信/QQ 沟通、当面交易
3. **优先做减法**，砍功能比加功能更有价值
4. **每个建议必须可落地**，给出具体代码/命令，不写伪代码
5. **保持极简**，能用一个文件解决的不拆两个，能用原生方法的不加库

### 对外部依赖零容忍
- 不引入第三方 API (通义千问、阿里云 SMS 等，除非用户明确要求)
- 不引入新的 npm 包 (除非确有必要且体积小)
- 不新增 Docker 容器
- 不新增数据库 (只用 MongoDB)

## 待办 / 已知不足

1. **HTTP-only Cookie**: 当前 token 存 localStorage，存在 XSS 风险。需 HTTPS 后迁移
2. **图片优化**: 前端未做压缩/缩略图，大图影响加载速度
3. **MongoDB 文本索引中文分词**: `$text` 对中文支持差，可能需要换策略
4. **无 HTTPS**: 当前 HTTP 明文传输(含 token)，需 Let's Encrypt
5. **无监控告警**: 没有 CPU/内存/磁盘/错误率监控
6. **单点故障**: 一台服务器，挂了就全挂
7. **冷启动**: seed.js 有 24 条种子数据但无真实图片，首页可能是空的
