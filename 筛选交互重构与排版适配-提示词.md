## 角色
你是电商 UI/UX 交互设计师，擅长 C2C 交易平台的筛选交互与布局适配。

## 任务
将我的商品列表页从「左侧筛选栏 + 右侧商品列表」的布局，改造为「顶部操作栏 + hover 浮层筛选 + 全宽商品列表」的交互模式。

---

## 一、当前现状

### 布局
```
md:flex-row
├── Filters (md:w-1/3 lg:w-1/4)  ← 左侧栏，占 25%~33% 宽度
│   全部是 <select> 下拉框
└── ProductList (w-full)          ← 剩余空间
     grid-cols-2 sm:2 lg:3 xl:4
```

### Filters 内部结构
| 筛选项 | 交互方式 | 说明 |
|--------|---------|------|
| 学院 | `<select>` 下拉 | 选项：全校所有学院 |
| 专业 | `<select>` 下拉（联动） | 选了学院才启用，选项随学院切换 |
| 分类 | `<select>` 下拉 | 9个固定分类 |
| 排序 | `<select>` 下拉 | 最新/低价/高价/最近 |
| 价格 | 双滑块 + 两输入框 + 确认按钮 | 最复杂的筛选 |

筛选「重置全部」按钮在底部。

---

## 二、改造目标

### 交互改造
把筛选从「侧边栏下拉框」改为「商品列表上方的顶部操作栏 + hover 浮层展开」：

```
[ 学院 ▼ ] [ 分类 ▼ ] [ 排序 ▼ ] [ 价格 ▼ ] [ 重置全部 ]
```

各筛选项的具体要求：

1. **学院** — hover 展开浮层显示所有学院；hover 到某个学院上，右侧再展开二级浮层显示该学院的专业（级联交互，类似京东分类导航）
2. **分类** — 与学院同级，hover 展开选项列表，点即选（无二级）
3. **排序** — 与学院同级，hover 展开选项列表，点即选（无二级）
4. **价格** — 与其他筛选同级，hover 展开浮层面板，内含最低价/最高价输入框 + 确认/重置按钮（精简版，去掉滑块）
5. **重置全部** — 独立按钮，和筛选项平级

需要给出这 5 个项在顶部操作栏的具体排列方案。

### 排版适配
筛选移走后，左侧栏空间全部释放，商品列表由「剩余宽度」变为「全宽」：

- 现有的 `grid-cols-2 sm:2 lg:3 xl:4` 是否需要调整列数？
- 全宽情况下，左右是否留白？留多少？
- 卡片尺寸是否需要等比调整？
- 桌面端（xl）空间变多后，4列会不会显得稀疏，是否需要直接上5列或6列？

### 移动端兜底
- 顶部操作栏在移动端如何展示？
- hover 在移动端不生效，降级为 click 展开

## 三、输出要求

给出 **2 套完整方案**，每套方案必须包含：

1. 顶部操作栏的布局排列（各按钮的顺序、间距、样式）

3. 筛选移走后商品列表的列数调整方案
4. 移动端降级方案
