# 2026-05-21 日志

## 修复：¥NaN 价格显示异常（Decimal128 序列化问题）

**根因**：MongoDB 中 price 字段为 Decimal128 类型，虽在 Product 模型定义了 toJSON/toObject transform，但在 populate 场景（购物车）和部分路径下 transform 不触发，前端 `parseFloat()` 得到 NaN。

**修复方案：双保险**
1. **后端显式转换** — 所有返回产品的接口都先 `toObject()` 再 `Number(productObj.price) || 0`
2. **前端兜底** — 所有 `parseFloat(product.price).toFixed(2)` 改为 `Number(product.price ?? 0).toFixed(2)`

**修改的文件**：

### 后端（强制价格显式转为数字）
- `Server/controllers/productController.js`：
  - `getAllProducts`：map 中加 `productObj.price = Number(productObj.price) || 0`
  - `getProductById`：改用 `toObject()` + 显式转换
  - `getProductsByUser`：map 转换价格
  - `getPurchasedProducts`：map 转换价格
  - `updateProductById`：转换返回的价格
  - `purchaseProduct`：转换返回的价格
- `Server/controllers/cartController.js`：
  - `getCart`：populate 后 `item.toObject()` 再 `Number(obj.productId.price)`
  - `checkoutCart`：success 结果中 `Number(product.price)`

### 前端（parseFloat → Number + ?? 0 兜底）
- `Client/src/components/Utility/ProductCard.js`：¥{Number(product.price ?? 0).toFixed(2)}
- `Client/src/components/Product_Details/ProductDetails.js`：同上
- `Client/src/components/Cart.js`：总价计算 + 商品价格（两处）
- `Client/src/components/UserProfile.js`：购物车标签页价格
- `Client/src/components/Profile/ProductList.js`：商品列表价格
