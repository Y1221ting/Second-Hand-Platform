# 搜索体验优化方案

## 概述
针对首页 `/home` 搜索链路的 UX 审查报告，覆盖「搜索 → 浏览 → 重新搜索」全流程。

## 页面布局（搜索→结果路径上的阻挡层）

```
Navbar (relative, 非 sticky)        ← ~56px
Announcement modal                  ← 全屏弹窗(首次访问)
HomeBanner                          ← ~150px
求购 banner                         ← ~50px
Recommendations (同学院推荐)        ← ~200-300px
───────────────── 分割线 ─────────────
"最近上新" / ProductList            ← 真正的内容
```

搜索后用户需手动滚过约 **500-600px** 的非商品内容才能看到结果。

---

## P0 — 严重影响体验，必须修

### 1. 搜索后页面不自动滚动到商品列表位置

**现状**：Navbar 搜索 → URL 更新 → `fetchProducts()` → 数据加载渲染 → scroll 位置保持在页面顶部。

**方案**：
- 文件：`Client/src/components/Home/index.js`
- 给"最近上新"标题加 `useRef` + `scrollIntoView`

```jsx
const productListRef = useRef(null);

useEffect(() => {
  if (search) {
    setTimeout(() => {
      productListRef.current?.scrollIntoView({ behavior: "smooth", block: "start" });
    }, 100);
  }
}, [search]);
```

- `index.css` 加 `html { scroll-behavior: smooth }`

### 2. Navbar 非 sticky → 搜索栏随滚动消失

**现状**：`<nav class="relative ...">`，向下滚动浏览商品后导航栏（含搜索框）消失，需滚回顶部才能重新搜索。

**方案**：
- 文件：`Client/src/components/Utility/Navbar.js`
- `relative` → `sticky top-0 z-30`
- 子元素（移动端搜索面板、下拉菜单）保持 `z-50`，正常浮于 Navbar 之上
- 所有共用 Navbar 的页面（Home、WantedPage、Cart、AddProduct 等）全局受益

### 3. 移动端：回车后搜索面板立即关闭，用户无反馈

**现状**：
```jsx
onKeyDown={(e) => {
  handleSearch(e);
  if (e.key === "Enter") setIsMobileSearchOpen(false);  // 立即关闭
}}
```

回车后面板消失 → 页面不滚动 → 用户不确定搜索是否触发。

**方案**：
```jsx
if (e.key === "Enter") {
  handleSearch(e);
  setTimeout(() => setIsMobileSearchOpen(false), 600);  // 延迟关闭
}
```

配合 P0 #1 的自动滚动，让用户看到搜索词已确认。

## P1 — 建议修复

### 4. 翻页后 scroll 停在底部

**现状**：Pagination 在底部 → 用户滑到底部点"下一页" → scroll 位置不变 → 看到的是上一页底部 / 空白。

**方案**：
```jsx
const paginate = (pageNumber) => {
  applyFilters({ page: pageNumber });
  setTimeout(() => {
    document.querySelector(".product-list-container")?.scrollIntoView({ behavior: "smooth", block: "start" });
  }, 50);
};
```

### 5. 搜索结果无数量提示

**现状**：商品直接以 grid 渲染，没有"找到 X 件商品"总结行。

**方案**：在 `fetchProducts` 中额外保存 `totalCount`，在 ProductList 上方渲染摘要行：

```jsx
{products.length > 0 && (
  <p className="text-sm text-gray-400 px-4 mb-2">
    共 <span className="text-white font-medium">{totalCount}</span> 件商品
    {filters.search && <> · 关于 "<span className="text-yellow-400">{filters.search}</span>"</>}
  </p>
)}
```

### 6. 搜索 placeholder 在求购页未切换

**现状**：`placeholder="搜索商品..."` 硬编码，在 `/wanted` 页面也显示"搜索商品"。

**方案**：
```jsx
const placeholder = location.pathname === "/wanted" ? "搜索求购..." : "搜索商品...";
```

两处（桌面 + 移动端）引用此变量。

### 7. 筛选变更也应自动滚动到结果区

**现状**：用户切换筛选（学院/分类/价格）后 scroll 不变，可能停留在空白区域。

**方案**：在 `location.search` 变化时（非首次渲染）自动滚动到列表区：

```jsx
const isFirstRender = useRef(true);
useEffect(() => {
  if (isFirstRender.current) { isFirstRender.current = false; return; }
  const timer = setTimeout(() => {
    listRef.current?.scrollIntoView({ behavior: "smooth", block: "start" });
  }, 150);
  return () => clearTimeout(timer);
}, [location.search]);
```

## P2 — 锦上添花

### 8. 长列表"返回顶部"浮动按钮

**方案**：右下角浮动按钮，滚动超过一屏显示，点击滚回顶部。

```jsx
const [showScrollTop, setShowScrollTop] = useState(false);
useEffect(() => {
  const onScroll = () => setShowScrollTop(window.scrollY > 600);
  window.addEventListener("scroll", onScroll, { passive: true });
  return () => window.removeEventListener("scroll", onScroll);
}, []);

// 底部 Tab 栏 fixed bottom-0 高 ~56px，按钮 bottom-20 刚好在 Tab 栏上方
{showScrollTop && (
  <button onClick={() => window.scrollTo({ top: 0, behavior: "smooth" })}
    className="fixed bottom-20 right-4 z-40 w-10 h-10 bg-yellow-500 rounded-full shadow-lg ...">
    ↑
  </button>
)}
```

### 9. Announcement 弹窗确认

Announcement 在用户同意后已通过 `localStorage` 标记屏蔽，后续搜索/导航不会再触发。无代码改动。
