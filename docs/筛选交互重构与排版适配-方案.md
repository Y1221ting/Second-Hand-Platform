# 筛选交互重构与排版适配方案（细化）

> 选定方向：**方案A** — 横排操作栏 + 级联浮层，hover/click 双触发

---

## 核心交互模型

### 触发方式：hover + click 双触发

| 操作 | 行为 |
|------|------|
| 鼠标移到按钮（学院/分类/排序） | 展开浮层（peek 模式） |
| 鼠标移出浮层区域 | 200ms 后自动关闭 |
| 鼠标点击按钮（所有筛选项） | 展开浮层（锁定模式） |
| 锁定状态下点击同按钮 | 关闭浮层（toggle） |
| 锁定状态下点击外部 / ESC | 关闭浮层 |
| 锁定状态下点击浮层内部某个筛选项 | 选中该项，浮层不关闭（用户可能需要继续选） |
| 锁定状态下鼠标移出 | 浮层不关闭（锁定优先于 hover） |
| 价格按钮 hover | **不触发**——价格只能 click 触发 |

**关键逻辑：** click 展开的浮层「锁定」，hover 不再控制关闭。hover 展开的浮层「非锁定」，移出后自动关闭。用户 click 了浮层内部的任意元素，自动升级为锁定状态。

```js
// 伪代码
let isLocked = false;

onButtonHover → if (!isLocked) openPanel()
onPanelLeave  → if (!isLocked) closePanel()
onButtonClick → isLocked = !isLocked; togglePanel()
onPanelClick  → isLocked = true  // 锁定
onClickOutside / ESC → isLocked = false; closePanel()
```

## 1. 顶部操作栏布局

```
[ 🏫 全部学院 ▾ ]  [ 📦 全部分类 ▾ ]  [ ↕️ 最新发布 ▾ ]  [ 💰 价格 ▾ ]              [ 重置 ]
```

### 按钮状态机

| 状态 | 样式 | 说明 |
|------|------|------|
| 默认 | `text-gray-300 bg-transparent` | 无筛选 |
| hover | `text-white bg-gray-800` | 鼠标悬停 |
| 展开中 | `text-white bg-gray-800 border border-yellow-500/30` | 浮层打开中，箭头变为 ▴ |
| 已选中 | `text-yellow-400 font-medium` | 有筛选值，显示具体值 |

### 选中态按钮文字变化

- 学院：选了"信息学院" → `[ 🏫 信息学院 ▾ ]`；没选 → `[ 🏫 全部学院 ▾ ]`
- 分类：选了"电子数码" → `[ 📦 电子数码 ▾ ]`；没选 → `[ 📦 全部分类 ▾ ]`
- 排序：选了"价格从低到高" → `[ ↕️ 价格从低到高 ▾ ]`；默认 → `[ ↕️ 最新发布 ▾ ]`
- 价格：选了 ¥100-¥500 → `[ 💰 ¥100-¥500 ▾ ]`；没选 → `[ 💰 价格 ▾ ]`

### 重置按钮

- 只有有活跃筛选时才显示（`filters` 中任意非默认值），否则隐藏
- 但更常见的做法是一直显示，灰色文字，点击清除
- 我建议一直显示，但灰色文字，点击后清除所有

### 样式细节

```
bg-gray-900/50 backdrop-blur-sm rounded-lg px-3 py-2
  flex items-center gap-1.5
    button: px-3 py-1.5 text-sm rounded-md whitespace-nowrap
    button-active: bg-gray-800 text-white border border-yellow-500/30
    button-selected: text-yellow-400 font-medium
    reset: text-gray-500 hover:text-red-400 text-sm ml-auto
```

## 2. 筛选浮层交互细节

### 学院 — 二级级联浮层

```
点击/hover「学院」→ 展开
┌──────────────────────────────────┐
│  第一级 180px    │  第二级 220px   │
├──────────────────────────────────┤
│  全部学院         │               │
│  ─────────        │               │
│  信息学院      ▸  │  计算机        │  ← hover「信息学院」时右侧展开
│  计算机学院    ▸  │  软件工程       │
│  外国语学院    ▸  │  网络空间安全   │
│  电子工程学院  ▸  │  ...           │
│  机械工程学院  ▸  │               │
│  ...             │               │
└──────────────────────────────────┘
```

**第一级操作：**
- 点击「全部学院」→ 清除学院筛选，浮层关闭
- 点击学院名（如「信息学院」）→ 选中该学院（不选专业），浮层关闭
- hover 学院名（或点击 ▸ 区域）→ 右侧展开该学院的专业列表

**第二级操作：**
- 点击某个专业名 → 选中该学院+该专业，浮层关闭
- hover 到另一个学院 → 右侧专业列表切换
- 鼠标移到二级区域时，一级的高亮跟随

**容器定位：**
- 浮层为 `absolute`，定位在操作栏下方
- 左侧对齐操作栏的起始位置
- `z-40`，阴影 `shadow-xl`，背景 `bg-gray-900 border border-gray-700 rounded-lg`

### 分类 — 单层浮层

```
点击/hover「分类」→ 展开
┌──────────────────┐
│  全部             │
│  教材教辅         │
│  电子数码    ← 选中态高亮 │
│  生活用品         │
│  体育用品         │
│  服饰美妆         │
│  文具办公         │
│  宿舍神器         │
│  乐器爱好         │
│  其他             │
└──────────────────┘
```

- 单列列表，hover 项有浅色背景
- 点击选中，选中的项有黄色圆点或 `✓` 标记
- 点击同一项 = 取消选择（恢复为"全部分类"）
- 点击「全部」清除分类筛选

### 排序 — 单层浮层

```
点击/hover「排序」→ 展开
┌──────────────────┐
│  ◉ 最新发布       │  ← 当前选中（黄点）
│  ○ 价格从低到高   │
│  ○ 价格从高到低   │
│  ○ 离我最近       │
└──────────────────┘
```

- 单选，黄点标记当前选中
- 点击切换
- 和现有行为一致：`handleSortChange`

### 价格 — 纯 click 触发小弹窗（与学院/分类/排序区分）

价格筛选**不响应 hover**，只能 click 触发——因为输入操作需要稳定聚焦，hover 展开/关闭会打断输入。

#### 触发形态

```
默认：      [ 💰 价格 ▾ ]
已设置：    [ 💰 ¥100 - ¥500 ▾ ]    ← 显示当前价格范围，黄色字体
```

#### 弹窗布局

```
click「价格」→ 弹出小窗（固定，不随滚动移动）
┌──────────────────────────────┐
│  价格范围                      │  12px 灰色标题，字号 text-xs
│                              │
│  ┌────────┐  ──  ┌────────┐  │
│  │  0     │      │  10000 │  │  输入框 bg-gray-800, text-white
│  └────────┘      └────────┘  │  text-center, w-24
│   最低价           最高价      │  标签在输入框下方，text-gray-500 text-xs
│                              │
│  ┌──────────┐ ┌──────────┐   │
│  │   确认    │ │   重置    │   │  按钮等高 32px
│  └──────────┘ └──────────┘   │  确认 bg-yellow-500, 重置 bg-gray-700
└──────────────────────────────┘
   ▲ 圆角 rounded-xl
   ▲ 阴影 shadow-2xl
   ▲ 背景 bg-gray-900 border border-gray-700
   ▲ 宽度固定 260px
```

#### 弹窗交互细节

**打开/关闭：**
- click「价格」按钮 → 弹窗打开，自动聚焦到最低价输入框
- click 外部区域 / ESC → 弹窗关闭，不保存当前输入
- 打开其他筛选浮层 → 价格弹窗先关闭
- click 同按钮 → toggle 关闭

**输入与确认：**
- 输入框 `type="number" min="0" max="10000"`，仅数字
- 空输入框视为 0（最低价）或 10000（最高价）
- 确认时校验：如果最低价 > 最高价，交换两者值（不弹 alert，静默修正）
- 确认后：调用 `handlePriceRangeChange(min, max)`，弹窗关闭，按钮文字变为 `💰 ¥min - ¥max ▾`
- 重置后：调用 `handlePriceRangeChange(0, 10000)`，弹窗关闭，按钮文字恢复 `💰 价格 ▾`

**弹窗内按 Enter → 触发确认**（键盘快捷操作）

**三态展示：**

| 状态 | 按钮外观 | 弹窗输入值 |
|------|---------|-----------|
| 未设置 | `💰 价格 ▾`（灰色） | 最低价=空，最高价=空 |
| 已设置 | `💰 ¥100 - ¥500 ▾`（黄色） | 最低价=100，最高价=500 |
| 展开中 | `💰 价格 ▴`（箭头翻转） | 显示当前设置值（如有） |

#### 和现有价格参数同步

弹窗打开时，从 URL 参数中读取当前 `minPrice`/`maxPrice` 填入输入框。弹窗是展现实时状态，不做中间缓存。

```js
// 弹窗打开时
const params = new URLSearchParams(location.search);
setMinInput(params.get("minPrice") || "");
setMaxInput(params.get("maxPrice") || "");
```

### 浮层通用行为

- 一次只有一个浮层打开——打开新浮层自动关闭旧浮层
- 点击浮层内部任意地方 → 锁定该浮层
- ESC 关闭当前浮层
- 点击外部区域（`mousedown` 事件）关闭当前浮层
- 所有浮层 `overflow-y-auto`，最大高度不超过视口 60%

## 3. 商品列表列数调整

筛选栏释放后，商品列表变全宽：

```jsx
// 改前
<div className="grid grid-cols-2 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-3 lg:gap-4">

// 改后
<div className="grid grid-cols-2 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-5 gap-4 lg:gap-5">
```

| 断点 | 改前列数 | 改后列数 | 预估卡片宽度 |
|------|---------|---------|------------|
| 默认（<640px） | 2 | 2 | ~160px |
| sm（≥640px） | 2 | 2 | ~220px |
| lg（≥1024px） | 3 | 3 | ~250px |
| xl（≥1280px） | 4 | **5** | **~230px** |
| 2xl（≥1536px） | 4 | **5**（保持） | **~240px** |

> 为什么 xl 直接上 5 列？当前 4 列时 xl 下每张卡约 280px，释放筛选栏后卡宽会到 340px+，太宽了，浏览时需要转头。5 列保持 230-240px 是最舒适的阅读宽度。

### 容器宽度

```jsx
// 改前
<main className="lg:w-4/5 mx-4 md:mx-auto py-4">

// 改后
<main className="max-w-7xl mx-auto px-4 md:px-6 py-4">
```

- `max-w-7xl` = 1280px，与当前 `lg:w-4/5` 在 1440px 屏上算出的宽度相近
- 左右 padding：移动端 16px，桌面 24px
- 容器居中 `mx-auto`

### SkeletonCard 同步

列数变化后，骨架屏 grid 同步更新：

```jsx
// 骨架屏改后
<div className="grid grid-cols-2 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-5 gap-4 lg:gap-5">
```

## 5. 与现有代码的兼容

### 数据流不变

现有 URL 参数驱动的筛选逻辑完全保留：

```js
// Home/index.js 中的 applyFilters 函数不改动
const applyFilters = (updates) => {
  const current = new URLSearchParams(location.search);
  for (const [k, v] of Object.entries(updates)) {
    if (v === "" || v === null || v === undefined) current.delete(k);
    else current.set(k, String(v));
  }
  if (!("page" in updates)) current.delete("page");
  const qs = current.toString();
  navigate(qs ? `/home?${qs}` : "/home", { replace: true });
};
```

所有筛选变更仍然调用 `applyFilters`，只是触发方式从 `<select onChange>` 变为「浮层中点击选项后调用」。业务逻辑零改动。

### ActiveFilterTags 保留

展示当前活跃筛选标签的 `ActiveFilterTags` 组件保持不变，放在顶部操作栏和商品列表之间。

### 价格去除滑块

不再需要 `rc-slider` 依赖，但初始不删除（可能其他组件在用？先确认无引用后再移除）。

## 7. 风险点

- **hover 延迟 200ms 的体验**：太短容易误关，太长觉得卡顿。需要实测调整。
- **锁定状态管理**：hover/click 双触发 + 锁定状态，`useState` + `useRef` 管理闭包需要细心。
- **浮层定位**：`absolute` 相对于操作栏容器，需要确保操作栏的 `position: relative`。滚动时浮层不能错位。
- **移动端 sheet 滚动穿透**：底部 sheet 打开时，页面主体不应滚动，需要 `overflow: hidden` 锁 body。
- **性能和重新渲染**：每个筛选按钮 hover 时展开/关闭不应触发 ProductList 重新渲染。
