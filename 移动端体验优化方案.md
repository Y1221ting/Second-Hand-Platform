# 移动端体验优化方案

> 撰写日期：2026-06-08
> 当前进度：第一期 ✅ / 第二批 ✅ / 第三批待开始

---

## 问题总览（按优先级排序）

### 🔴 问题 1：管理后台侧边栏在手机上完全不可用
- **文件**：`Admin/AdminLayout.js`
- **严重度**：影响功能
- **现状**：侧边栏固定 56px 宽，无移动端适配，手机上内容区被严重挤压
- **方案**：汉堡菜单折叠 + 滑出覆盖层 + 过渡动画
- **状态**：✅ 已完成

### 🟡 问题 2：购物车列表项在手机上信息过载
- **文件**：`Cart.js`
- **严重度**：影响体验
- **现状**：图片 + 名称 + 单价 + 数量按钮 + 金额 + 删除全部挤在一行
- **方案**：双行布局，第一行图片+信息+删除，第二行数量控制+金额
- **状态**：✅ 已完成

### 🟡 问题 3：首页筛选面板切换生硬
- **文件**：`Home/Filters.js`
- **严重度**：影响体验
- **现状**：点击「筛选」瞬间展开/关闭，无过渡动画
- **方案**：`max-h` 过渡动画（`transition-all duration-300`）
- **状态**：✅ 已完成

### 🟡 问题 4：商品卡片按钮触控区域偏小
- **文件**：`Utility/ProductCard.js`
- **严重度**：影响体验
- **现状**：按钮 `py-1.5 text-xs`，触控高度 ~28px（iOS 推荐 44px）
- **方案**：`py-1.5` → `py-2.5`，触控区域提高到 ~36px
- **状态**：✅ 已完成

### 🟡 问题 5：用户主页 6 个 Tab 占用过多垂直空间
- **文件**：`UserProfile.js`
- **严重度**：影响体验
- **现状**：6 个 Tab 折行 3 行，首屏看不到商品内容
- **方案**：`overflow-x-auto` 横向滑动，不折行
- **状态**：✅ 已完成

### 🟢 问题 6：Footer 内边距过大
- **文件**：`Utility/Footer.js`
- **严重度**：轻微不适
- **现状**：`p-12` 在手机上造成大量空白
- **方案**：`p-6 md:p-12`
- **状态**：⏳ 待做

### 🟢 问题 7：部分页面缺少返回按钮
- **文件**：`AddProduct.js`、`UserProfile.js` 等
- **严重度**：轻微不适
- **现状**：部分页面没有顶部返回按钮，移动端依赖浏览器后退
- **方案**：统一页面标题栏模式（Cart.js 已做对）
- **状态**：⏳ 待做

### 🟢 问题 8：Landing 页背景图加载慢
- **文件**：`Landing.js`
- **严重度**：轻微不适
- **现状**：加载外部大图 ~2MB，未做响应式适配
- **方案**：使用 CSS `@media` 切换移动/桌面尺寸，或改为纯色渐变背景
- **状态**：⏳ 待做
